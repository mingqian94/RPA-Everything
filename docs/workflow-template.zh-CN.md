# 任务描述模板

把下面这段复制给 Agent，再把括号里的内容替换成你的真实流程。

```text
我要把一个重复工作做成 RPA Skill。请先 dry-run 规划，不要真正点击最终提交/发布/发送。

任务名称：
（例如：每天导出 CRM 今日跟进客户）

我要操作的系统：
（网页地址 / 桌面软件名 / Android App / iPhone 半自动）

开始前我会准备好：
（例如：Chrome 已登录；手机已连接；素材文件放在哪个文件夹）

完整人工步骤：
1. 打开……
2. 点击……
3. 输入/选择……
4. 检查……
5. 最后一步是……（如果会发消息、审批、付款、删除，请明确写“需要我确认后再做”）

成功标准：
（例如：下载到一个 Excel；页面显示“提交成功”；手机停在发布前确认页）

不要做的事：
（例如：不要真实发布；不要删除数据；不要把账号、客户信息、截图写进代码）

希望保存成：
skills/（给这个流程起一个英文名，例如 crm/export_today_followups.py）
```

推荐执行顺序：

```bash
python run.py harness/agent -- --goal "把上面的任务先规划一下" --dry-run
python run.py harness/agent -- --goal "按上面的步骤跑通一次，但最后提交/发布前停住" --export skills/my_workflow.py
python run.py skills/my_workflow
```

如果任务会发消息、审批、发布、付款、删除远端数据，正式执行前必须加人工确认；不要在第一次探索时直接做最终动作。
