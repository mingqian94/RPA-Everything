# 外部操作确认单

用于发送、发布、审批、付款、删除、修改远端数据等不可逆或会影响他人的任务。首次探索和调试时，不应执行最后一步。

## 填写内容

```text
任务名称：
执行的 Skill：
执行环境和账号：

本次允许的最终操作：
（例如：向指定群聊发送一条已审核的通知；不要写“处理全部消息”）

对象和范围：
（收件人/页面/订单/文件/时间范围；不要把密码、Cookie、API Key 写进这里）

允许执行的次数：
成功的可见信号：
失败后该怎么停下：
是否可撤回；如可撤回，撤回方式：

确认人：
确认时间：
确认口令：我已核对目标、范围和内容，允许执行上述一次最终操作。
```

## 执行规则

1. 没有明确确认口令时，Skill 必须停在最终提交、发送、发布、审批、付款、删除前。
2. 确认只覆盖本单中写明的目标、范围和次数；范围变化后要重新确认。
3. Harness 探索任务必须显式传入 `--confirm-external` 才能执行外部提交类步骤；该参数不替代本确认单。
4. 执行后保存最小必要的结果证据，不保存账号、密钥、Cookie、客户隐私或完整敏感截图。

对应的完整边界见 [SECURITY.md](../SECURITY.md)。
